Why locca

Defaults that respect your hardware.

Most local-LLM tools either hide llama.cpp behind a wrapper or expose every flag at once. locca picks defaults that actually work on shared-VRAM iGPUs — and gets out of your way.

⌁

Tuned out of the box

Vulkan, --flash-attn on, q8_0 KV cache, --parallel 1, --batch-size 1024, --jinja. A 7–9B model at 128k context fits on a 16 GB shared-VRAM iGPU.

⤬

Auto context per model

ctxForModel() reads the model name and picks a sane ctx — 128k for MoE & 7–9B, scaled down for dense 12–35B. One regex away from tuning for your VRAM.

⌘

Composes, doesn't capture

Detects attached and external llama-servers instead of fighting them for VRAM. Bring your own supervisor — locca will use it.

Surface

A small set of commands. Each one earns its place.

Run locca with no args for the menu, or jump straight to what you need.

locca piLaunch the pi coding agent against your local server.
locca serveStart llama-server with a picked model, detached.
locca switchStop current server, start a new model with pi.
locca benchRun llama-bench with a friendlier summary.
locca statusServer, llama.cpp, and models summary.
locca apiPrint OpenAI-compatible connection info + LAN URLs.
locca logsTail the server log (locca-spawned servers only).
locca downloadPull a GGUF from HuggingFace into your models dir.
locca searchFuzzy-search HuggingFace for GGUF models.
locca deleteRemove a model directory you no longer need.
locca stopStop the running server.
locca setupRe-run the setup wizard.

Get going

Install in two steps.

Install llama.cpp, then install locca. First run launches a wizard that finishes config for you — and prints the exact install hint for your distro if llama.cpp isn't on $PATH.

macOS · Homebrew
# 1. install llama.cpp
$ brew install llama.cpp
 
# 2. install locca
$ npm install -g locca
$ locca
Arch / EndeavourOS
# official pkg (CPU + Vulkan)
$ sudo pacman -S llama.cpp
 
# or AUR for cutting-edge
$ yay -S llama.cpp-vulkan-git
$ npm install -g locca
Debian · Ubuntu · Fedora · openSUSE · Alpine
# build llama.cpp from source —
# `locca setup` prints the exact
# apt/dnf/zypper/apk line you need.
 
$ npm install -g locca
$ locca # wizard runs, tells you what's missing
From source
$ git clone https://github.com/perminder-klair/locca.git
$ cd locca
$ npm install && npm run build
$ npm link # symlinks `locca` into your PATH

Bonus

One command into the pi coding agent.

locca pi qwen fuzzy-matches the first *qwen*.gguf in your models dir, brings up the server if it's not already running, and registers itself as a custom OpenAI-compatible provider in ~/.pi/agent/models.json. Switch model, switch brain — locca switch llama-3.1-70b.
